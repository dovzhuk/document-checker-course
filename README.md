# Document Checker Course Project

Прототип системы автоматизированной проверки документов для курсового проекта.

## Идея проекта

Система принимает изображение документа или PDF, извлекает текст с помощью OCR, обрабатывает его и применяет модель машинного обучения для получения итогового результата.

Основная прикладная задача проекта — классификация типа документа по тексту, полученному после OCR.

## Цель

Разработать демонстрируемый CPU-only прототип, который можно показать на защите:
- загрузка документа;
- OCR;
- предобработка текста;
- предсказание модели;
- вывод результата через простой интерфейс.

## Структура проекта

- `data/` — данные проекта:
  - `raw/` — исходные файлы датасета и документов;
  - `interim/` — промежуточные данные;
  - `processed/` — подготовленные данные для обучения;
  - `ocr_eval/` — тестовый набор и результаты сравнения OCR.
- `notebooks/` — Jupyter Notebook для EDA, OCR-исследования и экспериментов с ML.
- `src/document_checker/` — основной код проекта:
  - `ocr/` — запуск и обёртки OCR;
  - `preprocessing/` — очистка и подготовка текста;
  - `features/` — извлечение признаков;
  - `models/` — обучение и инференс ML-модели;
  - `pipeline/` — полный пайплайн обработки документа;
  - `api/` — служебный слой для интерфейса и/или backend.
- `apps/streamlit/` — Streamlit-приложение для локальной демонстрации.
- `artifacts/` — сохранённые артефакты:
  - `models/` — обученные модели;
  - `metrics/` — метрики и отчёты;
  - `figures/` — графики и визуализации.
- `scripts/` — вспомогательные скрипты.
- `tests/` — тесты.
- `configs/` — конфигурационные файлы.
- `demo/` — примеры документов для демонстрации.
- `slides/` — материалы для презентации.

## План реализации

1. Подготовка структуры проекта и воспроизводимого окружения.
2. Подготовка и разметка данных для OCR-исследования.
3. Сравнение 2–3 OCR-подходов по метрикам.
4. Подготовка текстового пайплайна и обучение ML-модели.
5. Сборка полного пайплайна `document → OCR → text → ML → result`.
6. Создание Streamlit-интерфейса для демонстрации.
7. Подготовка метрик, примеров и презентации.

## Запуск

Базовое окружение управляется через Poetry.

Пример запуска Streamlit-приложения будет добавлен после подготовки первой рабочей версии интерфейса.
